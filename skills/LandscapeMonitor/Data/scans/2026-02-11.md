## CRITICAL ALERTS

### 1. Claude Desktop Extensions 0-Click RCE (CVSS 10/10)
**Priority:** CRITICAL
**Disclosed:** February 9, 2026 (LayerX)
**Impact:** Direct Anthropic ecosystem risk

**Details:**
- Zero-click RCE in Claude Desktop Extensions (DXT) affects 50+ extensions and 10,000+ users
- MCP servers run with **full system privileges** on host — no sandboxing like Chrome extensions
- Attack vector: Malicious instructions embedded in Google Calendar events; Claude interprets vague prompts like "take care of it" as justification to execute arbitrary code
- **Anthropic declined to fix at this time**

**PAI Relevance:** We don't use Claude Desktop Extensions (we use Claude Code CLI), but the underlying MCP privilege model is shared. The "vague prompt → arbitrary execution" pattern is exactly the kind of prompt injection our ExternalContentValidator is designed to catch.

**Action Required:**
- Monitor for Anthropic's response — "declined to fix" may change under pressure
- This validates our zero-trust approach to external content in MCP contexts

### 2. GitHub Copilot RCE via Prompt Injection (Patch Tuesday)
**Priority:** CRITICAL
**Disclosed:** February 10, 2026 (Microsoft Patch Tuesday)
**Impact:** Developer tool supply chain

**Details:**
- 3 RCE vulnerabilities in GitHub Copilot across IDEs:
  - **CVE-2026-21523** — RCE in GitHub Copilot + VS Code
  - **CVE-2026-21516** — RCE in GitHub Copilot for JetBrains
  - **CVE-2026-21256** — RCE in GitHub Copilot + Visual Studio
- Root cause: Command injection triggered via **prompt injection** — tricking AI agent into executing malicious commands
- Developers are high-value targets (access to API keys, AWS/Azure creds)

**PAI Relevance:** This is the same attack class we defend against. Prompt injection → command execution is the primary threat vector for any agentic coding assistant. Validates our permission model and hook-based validation.
